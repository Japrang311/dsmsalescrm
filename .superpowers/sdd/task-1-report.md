### Implementation

- Added the pure `requireLocalSupabaseUrl()` guard. It accepts only exact `http` origins on `127.0.0.1`, `localhost`, or `[::1]`, always on port `54321`; credentials, paths, query strings, fragments, HTTPS, other ports, malformed URLs, and hosted URLs fail closed. There is no bypass environment option.
- `supabase/tests/helpers.ts` invokes the guard before constructing its service-role `adminClient`.
- Removed the fixed-password Super Admin Auth and profile rows from ordinary `supabase/seed.sql`. The original Manager/Sales/Executive local seed users remain unchanged.
- Added `supabase/scripts/bootstrap-local-super-admin.ts`. It invokes the same guard before constructing its service-role client, idempotently creates/updates only the disposable local Super Admin Auth/profile fixture, and cannot target a hosted origin.
- Made role fixture creation failure-atomic: every successfully created Auth ID is tracked and deleted in reverse order after a later Auth/profile failure. A rollback failure produces an `AggregateError` whose first entry remains the primary setup error.
- Made teardown accept undefined or partial fixtures, attempt every known Auth deletion, and surface all returned/thrown cleanup failures in an `AggregateError`.
- Added integration coverage for active privileged caller/inactive target visibility, inactive privileged caller denial, and missing-profile caller denial. Test cleanup reactivates/deletes fixtures while preserving a primary assertion error if cleanup also fails.
