## Implementation

- Recreated the SELECT/INSERT/UPDATE policies for `clients`, `tasks`,
  `commercial_items`, and `sales_orders` with the explicit matrix:
  - active Sales: owner-only read/write;
  - active Manager: company-wide read/write;
  - active Executive: company-wide read-only;
  - active Super Admin: company-wide supported read/write.
- Recreated `follow_up_logs` as company-readable for Manager, Executive, and
  Super Admin, with INSERT for Sales-own, Manager, and Super Admin. It remains
  append-only with no UPDATE/DELETE policy or authenticated privilege.
- Added Super Admin company read/write to `targets` while preserving Sales-own
  read, Manager write, Executive read-only, and no delete.
- Replaced `org_settings` `USING (true)` with an explicit active four-role read
  check. Manager and Super Admin can update; the singleton remains
  insert/delete-denied.
- Added Super Admin company-wide SELECT/INSERT to `activity_log`. No
  UPDATE/DELETE policy or authenticated privilege exists.
- Left `profiles` Data-API SELECT-only. No profile write policy or privilege was
  added.
- Reasserted RLS on all nine current public tables. The
  `revenue_recognized` view remains `security_invoker=true` and inherits
  `sales_orders` RLS.
- Replaced implicit owner checks with the active-aware shape
  `current_user_role() = 'sales' AND owner_id/sales_id = auth.uid()`. This
  closes old-token access for inactive Sales users.
- Used scalar subqueries around `current_user_role()` and `auth.uid()` so the
  policy matrix does not trigger `auth_rls_initplan` performance warnings.
- Reasserted explicit relation privileges: anon has no public relation DML;
  authenticated receives only supported operations and no DELETE privilege.
- Replaced table-wide UPDATE grants on `clients`, `tasks`, `commercial_items`,
  and `sales_orders` with explicit column UPDATE grants that exclude
  `owner_id`. Sales, Manager, and Super Admin retain supported business
  corrections, but only the protected ownership-transfer path from the later
  lifecycle task can receive ownership authority.
- Preserved `owner_id` in every Super Admin correction exercised by the tests;
  direct owner reassignment is denied on all four owner-bearing mutable tables.
  No ownership-transfer function or lifecycle action was introduced.
- Added no Activity Log columns, event kinds, account-lifecycle functions, UI,
  or application role changes.

### Implementation

- Added `private.is_active_business_owner(uuid)` in the non-exposed `private`
  schema. It is `STABLE`, `SECURITY INVOKER`, uses an empty `search_path`, and
  returns true only for active `sales | manager` profiles. `PUBLIC` and `anon`
  have no execute privilege; `authenticated` and `service_role` have the
  execute privilege required by the policies, while the non-exposed schema
  prevents a public Data API RPC endpoint.
- Applied the predicate to owner-bearing INSERT/`WITH CHECK` policies on
  `clients`, `tasks`, `commercial_items`, `sales_orders`, `follow_up_logs`,
  and `activity_log`, plus INSERT/UPDATE `WITH CHECK` on `targets`.
- Bound every Activity Log website insert to
  `actor_id = (select auth.uid())`.
- Replaced authenticated table-wide Activity Log INSERT with explicit column
  INSERT grants for the nine business-event columns. Authenticated callers
  cannot provide server-owned `id` or `created_at`; `service_role` retains its
  full table INSERT privilege.
- Expanded Data-API behavior tests for Sales own-positive/other-negative
  reads and writes, Manager company writes including Manager-owned clients and
  targets, every Executive write verb granted to website roles, supported
  Super Admin writes, four-role direct `owner_id` reassignment denial, and
  Manager/Executive `revenue_recognized` reads.
- Added negative destination tests for Executive, Super Admin, inactive Sales,
  and inactive Manager; added inactive-owner correction denial and target
  destination-update denial.
- Centralized all tracked business-row cleanup. Restoration and deletion
  inspect every Supabase error, attempt all registered steps, and aggregate
  cleanup failures while retaining a primary setup/assertion failure.
- Added normal btree indexes on `owner_id` for `clients`, `tasks`,
